## 任务清单

### T1：模式切换（M1）

**任务描述**：在右上角模式按钮区域，新增「探伤」按钮，点击后切换到探伤模式。

**难易度**：⭐ 简单

**预计时间**：1 小时

**具体步骤**：
1. 在 `index.html` 模式按钮区域，新增「探伤」按钮（HTML + CSS）
2. 新增 `MODE_PROBE = 'probe'` 模式常量
3. 修改模式切换逻辑，支持「探伤」模式
4. 切换后，保留现有 10 节点结构，只改变显示内容
5. 新增模式切换过渡动画（节点颜色渐变过渡）

**核验标准**：
- ✅ 点击「探伤」按钮，成功切换到探伤模式
- ✅ 现有树结构保留，节点显示正常
- ✅ 可以从「探伤模式」切回其他模式

**i18n**：需要覆盖按钮文字（中英文）

### T2：演示模式基础（M2 - 基础）

**任务描述**：实现演示模式基础框架，包括状态A/B 判断、演示脚本播放、可跳过逻辑。

**难易度**：⭐⭐ 中等

**预计时间**：3 小时

**具体步骤**：
1. 实现状态判断逻辑（是否接入代码、是否开始扫描）
2. 编写内置屎山代码示例（`demo-code.js`，包含 6 种问题类型）
3. 实现演示脚本播放逻辑（30-60 秒，无文字/语音）
4. 实现「可跳过」逻辑（有信号立即跳演示）
5. 实现随机映射逻辑（每次随机挑 2-8 个节点变红）

**核验标准**：
- ✅ 未接入代码时，切换到探伤模式自动播放演示
- ✅ 已接入代码但未扫描，依然播放演示
- ✅ 演示过程中，接入代码或点「开始扫描」，立即跳演示
- ✅ 演示脚本包含 6 种问题类型（循环依赖/高耦合/深嵌套/死代码/超大函数/缺少错误处理）

**i18n**：不需要（演示无文字）

### T3：代码接入（M3）

**任务描述**：实现「上传文件夹」功能，支持用户上传 JS/TS 项目文件夹。

**难易度**：⭐ 简单

**预计时间**：1 小时

**具体步骤**：
1. 在界面上新增「上传文件夹」按钮
2. 使用 `<input type="file" webkitdirectory multiple>` 实现文件夹选择
3. 读取文件夹内所有文件（保留目录结构）
4. 将文件内容存储到浏览器内存（临时变量）
5. 上传成功后，提示用户「代码已接入，请点击开始扫描」

**核验标准**：
- ✅ 点击「上传文件夹」按钮，成功弹出文件夹选择器
- ✅ 选择文件夹后，成功读取所有文件
- ✅ 上传成功后，界面提示「代码已接入」

**i18n**：需要覆盖按钮和提示文字（中英文）

---

### T4：代码规范检查（M3 - 增强）

**任务描述**：上传后、扫描前，自动检查文件结构是否符合规范（acorn 能解析的 JS/TS 代码）。

**难易度**：⭐ 简单

**预计时间**：1 小时

**具体步骤**：
1. 编写规范检查逻辑（检查是否有入口文件、文件扩展名是否为 .js/.ts）
2. 符合规范 → 允许开始扫描
3. 不符合规范 → 弹窗提示「检测到不符合要求的程序，请规范后再尝试提交」
4. 弹窗后等待 1 分钟，没有新操作自动回到演示模式
5. 等待中途用户上传新代码 → 立即取消等待，处理新代码

**核验标准**：
- ✅ 上传符合规范的 JS/TS 代码 → 允许开始扫描
- ✅ 上传不符合规范的代码 → 弹窗提示
- ✅ 弹窗后 1 分钟无操作 → 自动回到演示模式
- ✅ 弹窗后中途上传新代码 → 立即处理新代码

**i18n**：需要覆盖弹窗文字（中英文）

---

### T5：代码扫描基础（M4 - 基础）

**任务描述**：集成 acorn，在浏览器里做静态分析，生成代码质量数据。

**难易度**：⭐⭐⭐ 中等偏难

**预计时间**：4 小时

**具体步骤**：
1. 引入 acorn 浏览器版本（CDN 或打包）
2. 用 Web Worker 跑 acorn（不卡界面）
3. 实现静态分析逻辑，生成 `quality` 对象（coupling/nestingDepth/linesOfCode/cyclomaticComplexity/hasCircularDep/isDeadCode）
4. 将 `quality` 对象映射到 tasks.json 的 10 个节点（固定映射规则）
5. 处理语法错误（acorn 解析失败 → 弹窗让用户选「自动修复」或「取消」）

**核验标准**：
- ✅ 上传 JS/TS 代码后，点「开始扫描」成功启动扫描
- ✅ 扫描完成后，tasks.json 的 10 个节点都有 `quality` 对象
- ✅ 扫描过程中界面不卡（用 Web Worker）
- ✅ 代码有语法错误 → 弹窗让用户选

**i18n**：需要覆盖弹窗文字（中英文）

## 核验清单（明天逐条校对）

### 核心流程核验

- [ ] 切换「探伤模式」→ 播放演示脚本（30-60 秒，无文字/语音，可跳过）
- [ ] 上传文件夹 → 自动检查结构是否符合规范 → 符合则允许扫描
- [ ] 点击「开始扫描」→ 顶部进度条显示进度 → 可取消
- [ ] 扫描完成 → 弹窗提示 → 点「查看报告」→ 扶桑树节点变颜色
- [ ] 点击问题节点 → 内部 Monaco Editor 展开代码片段 → 可手动修改
- [ ] 点击「复制路径」→ 复制文件路径 + 行号 → 用户手动去 VS Code 找
- [ ] 修复代码后 → 点「重新扫描」→ 节点颜色动态过渡（红→黄→绿）
- [ ] 全绿后 → 自动提示「是否重新全面扫描」→ 点「是」→ 再次扫描全绿 → 弹窗「修复成功！是否结束？」
- [ ] 点「结束」→ 回到演示模式
- [ ] 报告导出 → 底部状态栏按钮 → 点击导出 Markdown 报告

### 边界情况核验

- [ ] 上传非 JS/TS 代码 → 弹窗提示「目前只支持 JS/TS」
- [ ] 上传代码有语法错误 → 弹窗让用户选「自动修复」或「取消」
- [ ] 扫描过程中点「取消」→ 立即停止，回到演示模式
- [ ] 扫描前检查不符合规范 → 弹窗提示 → 等待 1 分钟无操作 → 自动回到演示模式
- [ ] 页面关闭后重新打开 → 不保存扫描结果，要重新上传、重新扫描

### i18n 核验

- [ ] 所有新增 UI 文字都有中英文翻译
- [ ] 切换语言后，所有文字都正确更新

### 性能核验

- [ ] 扫描过程中界面不卡（用 Web Worker）
- [ ] 放大/缩小正常，不卡顿
